# IELTS Daily — Project Instructions

## What is this?
Web app for daily IELTS reading practice. Hono/TypeScript backend, SQLite (better-sqlite3), deployed on Fly.io. Built for a Spanish-speaking student learning English.

## Build & Run
```bash
npm run build        # tsc → dist/
npm run start        # node dist/index.js
npm run dev          # tsc + RUN_NOW=true node dist/index.js
```

## File Layout
```
src/
├── index.ts              # Hono server entry point
├── db.ts                 # SQLite database
├── dashboard.ts          # Admin dashboard routes
├── services/             # Business logic
└── templates/            # HTML templates
specs/                    # Source of truth — never modify during build
PROMPT_build.md           # Build mode prompt for loop.sh
PROMPT_plan.md            # Plan mode prompt for loop.sh
IMPLEMENTATION_PLAN.md    # Generated by plan mode
loop.sh                   # Build/plan orchestrator
```

## Code Rules
- TypeScript strict mode
- Hono for HTTP routing
- better-sqlite3 for storage (no ORMs)
- @anthropic-ai/sdk for AI content generation
- Resend for email delivery
- Student-facing content in accessible English; UI instructions in Spanish
- No unnecessary dependencies

## Git Workflow
After completing any task:
1. Stage relevant files (never `git add -A` blindly)
2. Commit with conventional format: `feat:`, `fix:`, `chore:`, `refactor:`
3. Push to remote: `git push`

Commit message format:
```
feat: short description

- Detail 1
- Detail 2

Task: [Task Title from IMPLEMENTATION_PLAN.md]
```

## Linear Integration
This project tracks work in Linear: team **Lucas-home**, project **IELTS**.

### After completing a task:
1. Find the matching Linear issue by title
2. Set its state to "Done"
3. If you discover new work not in the plan, create a Linear issue:
   - Team: `Lucas-home`, Project: `IELTS`, Assignee: `me`
   - Priority: 1=Urgent, 2=High, 3=Normal, 4=Low
   - Include description with files and acceptance criteria

### After planning:
1. For each task in IMPLEMENTATION_PLAN.md, ensure a matching Linear issue exists
2. Update statuses: NOT STARTED → "Todo", IN PROGRESS → "In Progress", DONE → "Done"
3. Set `blockedBy` relations matching the plan's dependency graph

### Linear conventions:
- Issue titles should match IMPLEMENTATION_PLAN.md task titles
- Always search before creating to avoid duplicates
- Use comments for blockers or context
