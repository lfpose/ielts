# Improvements v2 — Design, UX & Game Overhaul

**Source**: Visual audit of all pages + feedback session (2026-04-05)  
**Goal**: Make the app feel like a real newspaper you play, not a form you fill out.

---

## 1. Dashboard

### 1.1 Newspaper Layout — Multi-Column Grid

The dashboard currently renders as a single boring column. It should feel like a newspaper front page.

**Desktop layout (≥ 800px):**
```
┌─────────────────────────────────────────────┐
│            THE IELTS DAILY masthead         │
│         VOL. 1  ·  DATE  ·  ED. DIGITAL    │
├─────────────────────────────────────────────┤
│ Hola, [Name]                    🔥 12 days  │
├──────────────────────┬──────────────────────┤
│                      │                      │
│  TODAY'S TOPIC       │  STREAK + HEATMAP   │
│  (big headline +     │                      │
│   illustration)      │  Progress 0/5        │
│                      │  ▓▓▓▓▓▓▓░░░░░░░     │
│                      │                      │
├──────────────────────┴──────────────────────┤
│  EJERCICIOS DE HOY                          │
│  ┌────────────┐ ┌────────────┐ ┌──────────┐ │
│  │ Ex 1       │ │ Ex 2       │ │ Ex 3     │ │
│  └────────────┘ └────────────┘ └──────────┘ │
│  ┌────────────────────┐ ┌──────────────────┐│
│  │ Ex 4               │ │ Ex 5             ││
│  └────────────────────┘ └──────────────────┘│
├─────────────────────────────────────────────┤
│  EDICIONES ANTERIORES (archive, below fold) │
└─────────────────────────────────────────────┘
```

- Two-column layout for the top section (topic + right sidebar)
- Vertical column rule (1px `var(--muted)`) between columns
- Exercise cards in a 3+2 grid on desktop, single column on mobile
- Max width: 1000px for dashboard (wider than current 960px)

### 1.2 Topic Headline — Hero Treatment

The day's topic should feel like a front-page banner headline, not just a subtitle.

- Topic text: Playfair Display 900, 36px, full width
- Subkicker above it: Inter 600, 10px, uppercase, letter-spacing 4px: "TEMA DEL DÍA"
- Below topic: the ASCII/text illustration (already generated by content pipeline — display it prominently in a `<pre>` with monospace font, centered, 60-80 char wide)
- Article count: "5 ejercicios · ~20 min"

### 1.3 Streak Widget — More Prominent

Current: plain numbers "0 / 0" with tiny heatmap. Needs personality.

- Fire emoji 🔥 large (24px), next to streak number
- Streak number: JetBrains Mono 700, 48px, `var(--fg)`
- "días seguidos" label: Inter 500, 11px, uppercase, muted
- Longest streak below: smaller, "Récord: 21 días"
- If streak = 0: show motivational copy: "¡Empieza hoy!" (small, muted)
- Streak widget gets a warm amber tint background when streak > 7

### 1.4 Progress Bar — Thicker & Interactive

Current: invisible thin dashes.

- Height: 8px segments (not 2px)
- 5 individual pill segments with 4px gaps between them
- Each segment: 
  - Empty: `var(--muted)` with subtle border
  - Completed: filled with `var(--fg)` (dark) or green if all 5 done
  - Colors fill left to right as exercises are completed
- Below: "3 de 5 completados" text
- When all 5 done: progress bar turns fully green + tiny confetti burst

### 1.5 Exercise Cards — Differentiated by Type

Current: plain white boxes with a red type label and title. All look identical.

Each exercise type gets a visual identity:

| Type | Left accent color | Icon/symbol |
|------|------------------|-------------|
| Lectura Larga | `#1a1a2e` (deep navy) | 📖 or ¶ symbol |
| Lectura Corta | `#2d4a22` (forest green) | § symbol |
| Vocabulario | `#4a1942` (deep purple) | Aa |
| Completa Espacios | `#3d2200` (dark amber) | __ |
| Escritura | `#1a0000` (dark red) | ✎ |

Card design:
- Left colored border (4px solid accent color)
- Type label in the accent color
- Exercise title: Playfair Display 700, 18px
- Status badge right-aligned: "Disponible" (muted) or score badge ("4/5", green tint)
- Estimated time: small muted text "~8 min"
- Completed cards: subtle green-tinted background (`#f0f7f0`), score shown, checkmark

### 1.6 Heatmap — Standalone Section

Move heatmap out of the streak widget into its own section below the exercise cards.

- Full width, centered
- Section header: "ACTIVIDAD — ÚLTIMAS 16 SEMANAS"
- Month labels above the grid
- Day labels (L M X J V S D) on the left
- Cell size: 14px × 14px, 3px gap
- Tooltip on hover: "12 puntos · lunes 3 de marzo"
- Current day cell: outlined with red border

## 4. Exercise 3 — Vocabulary Match (Full Game Redesign)

The current implementation is a plain HTML table. This is the exercise that most needs to feel like a game.

### 4.1 Layout
- Two columns side by side (desktop) / scrollable single list with tap-to-pair (mobile)
- Words column (left): 6 word chips, each in a rounded card
- Definitions column (right): 6 definition cards, shuffled, smaller text

### 4.2 Word Cards
- Background: `var(--n100)` (light gray card)
- Word text: Playfair Display 700, 16px, centered
- Border: 2px solid `var(--muted)` default
- Selected state: border-color darkens, slight scale(1.02) transform
- Connected state: border-color = pair color, background = pair color at 15% opacity

### 4.3 Definition Cards
- Smaller, more text — Inter 400, 13px
- Same states as word cards
- Shuffled order on load (not alphabetical)

### 4.4 Pair Colors
6 soft accent colors for connected pairs (assign on connection, not predetermined):
```
Pair 1: #E8F4E8 (sage green) + border #2D6A4F
Pair 2: #E8F0F8 (soft blue) + border #1a4a7a
Pair 3: #F8F0E8 (warm amber) + border #7a4a1a
Pair 4: #F0E8F8 (soft purple) + border #4a1a7a
Pair 5: #F8E8F0 (soft rose) + border #7a1a4a
Pair 6: #E8F8F8 (soft teal) + border #1a7a6a
```

### 4.5 Connection Animation
When user connects a pair:
- Both cards simultaneously get their shared color (CSS transition 200ms)
- Slight "bounce" effect (transform scale 1.0 → 1.04 → 1.0)
- A small connecting line or indicator between them (or just shared color is enough)

### 4.6 Interaction Model
1. Tap a word → word card gets "selected" state (dark border, label "Seleccionado")
2. Tap a definition → both snap to pair color, connection made
3. Tap an already-connected card → both return to default (undo)
4. Can also tap definition first, then word
5. All 6 connected → Submit button enables

### 4.7 Mobile (Stacked)
On mobile, don't do two columns. Instead:
- Show all word cards as a horizontal scroll row at top
- Show all definition cards stacked below
- Tap word → it highlights → tap definition to connect
- Connected pairs shown with shared color

## 5. Exercise 4 — Fill the Gap (Improved)

### 5.1 Paragraph Display
- Larger text: Lora 400, 17px, line-height 1.9
- Blanks displayed as: `______(1)______` — underlined span with number, not bracketed
- Selected blank: highlighted with `var(--red)` underline + slight yellow background
- Filled blank: the word appears inline in bold, underlined, with accent color

### 5.2 Word Bank
- Chips below the paragraph, flex-wrap
- Each chip: rounded pill button (Inter 600, 13px, `border: 1.5px solid var(--fg)`)
- Used chip: grayed out, `opacity: 0.4`, cursor `not-allowed`
- Hover on available chip: background fills in (dark bg, white text)
- Animation: chip "flies" to the blank when selected (optional, CSS transition)

### 5.3 Interaction
Two tap model (same as current but clearer):
- Tap blank → blank highlights (selected state)
- Tap chip → fills the selected blank, chip grays out
- Tap a filled blank → word returns to bank (chip re-activates)
- OR: tap chip first → it gets "selected" → tap blank → fills it

### 5.4 Visual Layout
- Paragraph section has a subtle paper/card background
- Word bank section has a label: "BANCO DE PALABRAS" (uppercase Inter, 10px, muted)
- Clear visual separation between paragraph and word bank
