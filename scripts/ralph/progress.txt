## Codebase Patterns
- Templates export render functions returning HTML strings with inline <script> for client-side JS
- Exercise submissions POST JSON to /s/:token/exercise/:exerciseId, expect JSON response {score, maxScore, feedback}
- Grading in src/services/grading.ts — deterministic for exercises 1-4, AI-graded for writing
- DB queries in src/db.ts, routes in src/routes/, templates in src/templates/
- Design spec at specs/design.md — Playfair Display headings, Lora body, Inter UI, warm off-white palette
- Admin template uses CSS grid layout with sidebar (240px) + main content (1fr), section show/hide via vanilla JS
- Admin at specs/admin.md — clean functional design, Inter font, no newspaper aesthetic
- ESM project: all imports need .js extension in TypeScript files
- Client templates all POST { answers: { key: value } } objects; graders expect typed arrays — transformation lives in src/routes/student.ts
- Grader feedback is wrapped in { results: [...] } but templates expect flat arrays with different field names — transformation also in student.ts
- Route handler returns raw grader feedback to client (for reload) but stores transformed feedback in DB (for templates)
- Exercise content JSON: readings have {title, passage}, writing has {prompt}, vocabulary has {words[]}, fill_gap has {text/paragraph}
